# Beyond Functional Correctness: Exploring Hallucinations in LLM-Generated Code

## Metadata
**Authors**: Fang Liu, Yang Liu, Lin Shi, Zhen Yang, Li Zhang, Xiaoli Lian, Zhongqi Li, Yuchi Ma
**Topic**: Taxonomy, Causes, and Impacts of Code Hallucinations

## Problem Statement
Despite the rapid adoption of code LLMs, there is a lack of comprehensive understanding of the types, causes, and impacts of "hallucinations" specifically within code generation. Prior work largely focuses on natural language generation (NLG) or merges code hallucinations with general naive errors.

## Methodology
- **Thematic Analysis**: Collected 3,120 code samples generated by 4 representative LLMs (CodeLlama, DeepSeek-Coder, GPT-4, DeepSeek-R1) using 2 benchmarks (HumanEval, CoderEval).
- **Taxonomy Construction**: Manually annotated samples using an open coding process to construct a taxonomy of code hallucinations based on "explicit semantic conflicts with established facts."
- **Mitigation Exploration**: Experimented with prompt enhancement techniques like Self-Refine, Chain-of-Thought (CoT), and Retrieval-Augmented Generation (RAG) to reduce these hallucinations.

## Key Findings
- **Taxonomy**: Identified 3 primary categories of hallucinations:
  1. *Requirement Conflicting* (Faithfulness to Prompt)
  2. *Code Inconsistency* (Faithfulness to Generated Context)
  3. *Knowledge Hallucinations* (Factuality to Real-world Knowledge)
  These span 12 specific subcategories (e.g., Undefined Variables, Inconsistent Libraries).
- **Prevalence**: "Behavior Conflicting" (generating code that structurally defies the prompt's intended logic) is the most common hallucination across all evaluated models.
- **Root Causes**: 80.86% are model-related (inherent limitations), 26.24% stem from lacking domain-specific knowledge, and ~2% are from ambiguous prompts. Higher prompt complexity significantly increases hallucination rates.
- **Impacts**: The vast majority (95.30%) result directly in incorrect functionality, but they also severely degrade readability, maintainability, and computational efficiency.
- **Mitigation**: Simple training-free prompt enhancements (RAG for injecting missing repository context, and Self-Refine/CoT for reasoning) can substantially reduce hallucination rates.

## Limitations & Future Work
- **Limitations**: The manual coding process carries inherent subjectivity. The study was primarily constrained to Python and Java.
- **Future Work**: Proposes developing automated, LLM-driven mechanisms to detect hallucinations. Encourages using reinforcement learning to boost chain-of-thought capabilities and further integrating RAG for project-level code generation to eliminate knowledge gaps.
